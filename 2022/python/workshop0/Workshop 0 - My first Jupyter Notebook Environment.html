What‚Äôs a Jupyter?

Jupyter Notebook is a notebook environment used for running .ipynb (iPython Notebook) files. These files combine markdown blocks/cells (ie. text cells, that contain paragraphs, tables, LaTeX equations, bold text) as well as Python code blocks. These blocks can be run in any order (typically run top to bottom) with a memory of what was run before.

This allows you to intersperse discussion with the code that you used to generate your results (perfect for reports and reproducible work!).

This also makes it easy to tweak things (ie. to change my plot title, I don‚Äôt need to rerun my 30 minute simulation).

Compare this to normal .py files which:

  • Have limited ways of writting paragraphs (ie. only in comment blocks)
  • Everything runs top to bottom, and debugging often requires you to run the whole file again.

Lets set up a Jupyter Notebook environment! We have two options, an online environment or a local environment. Here‚Äôs a list of pros and cons of each to help you decide!:

  • Online
    • Pros
      • Its all online and backed up
      • Super easy to get started
      • Easy to collaborate with others
      • Can access GPUs for machine learning (not important for us)
    • Cons
      • You have to have internet to use the notebook
      • Can have high latency (as its an online environment) üò†üò†
        • No matter how simple your code block is, it will take an extra second to run (compared to a local environment which is instant).
      • Working with the filesystem is more complicated/annoying (linking up with Google drive etc)
  • Local
    • Pros
      • Super fast
      • Available regardless of if you have an internet connection
      • Means that your Jupyter Notebooks are just another file on your computer
    • Cons
      • Requires some setup (and some mild tech skills. ie, at least knowing how to install a program as per online instructions)
      • Collaboration with others is more difficult

Online environments

You can get away with using on online Jupyter Notebook environment with little downsides. If you want to use Python/Jupyter often, or get annoyed with things taking a while to load, I would highly recommend setting up a local environment.

For online Jupyter notebook environments, here are two:

  • Google Colab
  • CoCalc

Local environment (recommended)

You can also install Jupyter locally and run Jupyter notebooks. The best way to do this is through Anaconda.

Anaconda can be thought of as a Python installation with all the bells and whistles (ie. packages) necessary for scientific computing coming pre-installed. It also comes with a nice graphical interface which makes it easier to launch Jupyter.

Local Jupyter installations will also open in the browser, but you‚Äôll see in the search bar localhost:8888/tree or similar. The localhost indicates that it is connected to a local server running on your computer.

Basically Jupyter uses your web browser to display the application.

Note some UWA computers come with Jupyter installed. Search for ‚ÄúAnaconda Navigator‚Äù in the app search to check.

TLDR; Use Jupyter Lab if you can

You may have noticed in the GUI launch that there are two options ‚ÄúJupyter Notebook‚Äù or ‚ÄúJupyterLab‚Äù (or equivalently, jupyter notebook and jupyter lab for the terminal launch).

Whats the difference between the two? Jupyter Lab is the ‚Äúnext generation‚Äù user interface, and has some some more features (especially for window layout, and working with multiple workbooks at once). Jupyter Lab even has a feature where you can open the same notebook side by side (editting different parts of the notebook simultaneously, how nifty!).

Jupyter Lab is a more feature rich environment and allows you to edit more things than just Jupyter Notebooks. It also allows you to view your filesystem while you‚Äôre working.
